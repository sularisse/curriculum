# 開發紀錄與需求演進總結 (中國醫藥大學 課表產生器)

這份文件記錄了我們開發「中國醫藥大學 課表產生器」(`schedule.html`) 過程中的所有重要里程碑、遇到的問題以及解決方案。

## 1. 初始規劃與 UI 建立
- **需求**：建立一個包含多個下拉式選單（學年、學期、學制、開課系所、年級、修別）的 HTML 頁面。
- **功能**：使用者選擇條件後，向學校的課程資訊網 (`https://web1.cmu.edu.tw/courseinfo/`) 發送請求，抓取課程資料並顯示在下方的表格中。
- **挑戰**：前端直接呼叫學校 API 會遇到 CORS (跨來源資源共用) 阻擋。
- **解決方案**：導入免費的 CORS Proxy 服務（如 `api.codetabs.com`），成功跨域獲取 HTML 資料。

## 2. 課程清單與系所連動修復
- **問題**：一開始發現無法正確選取開課系所，或是跳出「查無課程資料，請調整篩選條件」。
- **原因**：表單參數未完全對齊學校後端要求的格式；且學校的 HTML 表格結構相當複雜，一開始的解析邏輯沒有對應到正確的表格。
- **解決方案**：重新觀察真實的 HTTP 請求參數，並修正 `parseCourseHTML` 函式，使其能精確鎖定包含資料的 `dataTable`。

## 3. 課表顯示與互動功能導入
- **需求**：
  - 不只要顯示清單，還要能在最下方勾選想要的課程。
  - 勾選後，將課程名稱放入位於畫面中間的「課表表格」(Timetable) 對應的時間格子中。
  - 在課表中的課程區塊上，增加一個「大綱」按鈕，點擊能直接連到該課的課程資訊頁面。
- **挑戰**：一開始勾選後，課表依舊空白沒有顯示。
- **原因**：學校網站的課表並沒有單一的「時間」欄位，而是採用 **七個獨立的星期欄位 (`ClassTable`)**，裡面填寫如 `12`、`雙(34)` 等節次。舊的解析器找不到時間欄，導致課程被判定為沒有排課時間。
- **解決方案**：全面重寫時間解析邏輯。改為遍歷這七個星期欄位，提取其中的節次，並將連續節次（如 `12`、`34`）正確切分。

## 4. 節次解析修正與「固定課程」功能
- **問題與需求**：
  - 節次 `12` 被誤判為「第1節」和「第2節」，但實際上學校的 `10`、`11`、`12` 代表的是夜間節次（第12節）。
  - 使用者希望更換篩選條件重新搜尋時，已經勾選的課程要能「固定」在課表中，不要消失。
  - 發現 115 學年度第一學期尚未有課程時程規劃，希望將課表連結強制導向 `114學年度第一學期`。
- **解決方案**：
  - 修正解析演算法：遇到字元 `0` 時，自動與前方的 `1` 合併成 `10`；其他如 `12`、`13` 亦正確處理。
  - 引入 `pinnedCourses` (Map 結構) 跨查詢保存狀態。將下方的清單分為「📌 已固定課程」與「🔍 查詢結果」兩個區塊。
  - 透過 `replace` 將抓取到的課程網址中的 `mSmtr` 參數強制替換為 `1141`。

## 5. 照片匯出與視覺排版優化 (跑版修復)
- **需求**：
  - 新增「下載課表照片」功能。
  - 若同一時段有兩門課（例如實驗課分單雙週），需將單週課程放左側、雙週放右側並加上標籤。
  - 去除課表格子內的教師名字，僅保留在滑鼠懸浮 (Tooltip) 時顯示。
  - 上課地點僅保留一個在有顏色的框框內，且字體適度放大（但小於課程名稱）。
- **挑戰與修復**：
  - 實作匯出功能時導入了 `html2canvas`，成功讓使用者能一鍵下載 PNG 圖片。
  - 為了讓單雙週並排，曾試圖將表格儲存格 (`td`) 設為 `display: flex`，導致了**嚴重的 HTML 表格跑版**。
  - **最終解決方案**：將 `display: flex` 改套用於儲存格「內部」的容器 `coursesContainer`，完美恢復原本的課表結構。同時調整 CSS，讓教室地點 (`.course-room`) 回歸各自的顏色方塊中，並設定適當字體大小 (名稱 13px，教室 11px，教師隱藏)。

目前所有功能（查詢、固定、跨域、單雙週並排、照片下載）皆已穩定運作！
