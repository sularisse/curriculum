# 開發紀錄與需求演進總結 (中國醫藥大學 課表產生器)

這份文件記錄了我開發「中國醫藥大學 課表產生器」(`schedule.html`) 過程中的所有重要里程碑、遇到的技術問題、挑戰，以及最終的解決方案。

## 1. 初始規劃與 UI 建立
- **需求**：建立一個包含多個下拉式選單（學年、學期、學制、開課系所、年級、修別）的 HTML 頁面，提供使用者完整的課程篩選條件。
- **功能**：使用者選擇條件後，向學校的課程資訊網 (`https://web1.cmu.edu.tw/courseinfo/`) 發送 HTTP 請求，動態抓取課程資料並顯示在下方的課表與清單中。
- **視覺與設計 (UI/UX)**：
  - 採用現代化的 **Glassmorphism (玻璃擬物化)** 設計風格，使用 `backdrop-filter: blur()` 與半透明背景。
  - 加入動態漸層背景動畫 (`@keyframes bgPulse`)，提升整體視覺質感。
  - 實作響應式佈局 (RWD)，確保在手機與平板上選單與課表也能正常閱讀。
- **挑戰**：前端直接發送 `fetch` 呼叫學校 API 會遇到嚴格的 CORS (跨來源資源共用) 阻擋。
- **解決方案**：
  - 導入免費的 CORS Proxy 服務，並建立 **Proxy 輪詢機制 (Fallback)**。
  - 程式中內建多個 Proxy 來源（如 `api.allorigins.win`, `corsproxy.io`, `cors.eu.org`, `proxy.corsfix.com` 等），當其中一個失效或超時，會自動切換至下一個嘗試，大幅提高資料抓取的成功率與穩定性。

## 2. 課程清單與系所連動修復
- **問題**：一開始發現無法正確選取開課系所，或是經常跳出「查無課程資料，請調整篩選條件」。
- **原因**：表單請求參數未完全對齊學校後端要求的長度與格式；且學校的 HTML 表格結構相當複雜，一開始的 DOM 解析邏輯沒有對應到正確的表格。
- **解決方案**：
  - 將學制與系所進行「連動綁定」，預先把大學部、碩士班、博士班等各系所的對應 Value 寫入 `DEPARTMENTS_DATA`，當切換學制時動態更新系所選單。
  - 重新觀察與對齊真實的 HTTP 請求參數（補齊 `Cos_setyear_q`, `Cos_setterm_q` 等幾十個欄位）。
  - 強化 `parseCourseHTML` 函式的強健度：優先搜尋含有 `.ClassTable` 的表格，若無則降級尋找含有「課程名稱」或「課號」表頭的表格，精確鎖定資料列。

## 3. 課表顯示與互動功能導入
- **需求**：
  - 不只要顯示清單，還要能在最下方清單勾選想要的課程。
  - 勾選後，將課程名稱動態放入位於畫面中間的「課表表格」(Timetable) 對應的星期與節次格子中。
  - 在課表中的課程區塊上，增加一個「時程」按鈕，點擊能直接連到該課的課程資訊與大綱頁面。
  - 滑鼠懸浮於課程區塊時，顯示包含教師、學分、完整時間、教室的 **Tooltip 提示框**。
- **挑戰**：一開始勾選後，課表依舊空白沒有顯示。
- **原因**：學校網站的課表並沒有單一的「時間」欄位，而是採用 **七個獨立的星期欄位 (`ClassTable` 星期日到星期六)**，裡面填寫如 `12`、`雙(34)` 等節次。舊的解析器找不到時間欄，導致課程被判定為沒有排課時間。
- **解決方案**：
  - 全面重寫時間解析邏輯 (`extractPeriodsFromCell`)。改為自動遍歷這七個星期欄位。
  - 實作 Tooltip 的滑鼠追蹤 (`mousemove`)，讓詳細資訊跟隨滑鼠顯示，同時避免超出螢幕邊界。
  - 實作自動分配課程顏色 (`COLORS` 陣列)，確保同一門課在不同時段視覺一致。

## 4. 節次解析修正與「固定課程」功能
- **問題與需求**：
  - **節次解析錯誤**：節次 `12` 被誤判為「第1節」和「第2節」，但實際上學校的 `10`、`11`、`12` 代表的是夜間節次（第12節）。其他還有 `1-4` 或 `1,2,3` 等不同表示法。
  - **狀態保留**：使用者希望更換篩選條件重新搜尋時，已經勾選的課程要能「固定」在課表中，不要因為重新查詢就消失。
  - **時程連結問題**：發現 115 學年度第一學期尚未有課程時程規劃，希望將課表連結強制導向 `114學年度第一學期`。
- **解決方案**：
  - **強化正則與節次拆解**：支援 `1-4` 區間展開、`,` 逗號分隔。遇到連續字串時，若遇到字元 `0`，會自動與前方的 `1` 合併成 `10`；其他如 `12`、`13` 亦正確當作獨立字元處理。
  - 引入 `pinnedCourses` (Map 結構) 跨查詢保存狀態。將下方的清單分為「📌 已固定課程」與「🔍 查詢結果」兩個區塊，並在重新查詢時自動將已固定課程的狀態打勾。
  - 透過正則表達式 `.replace(/mSmtr=\d+/, 'mSmtr=1141')`，將抓取到的課程網址參數強制替換為 `1141`，確保大綱連結不失效。

## 5. 照片匯出與視覺排版優化 (跑版修復)
- **需求**：
  - 新增「下載課表照片」功能。
  - **單雙週並排**：若同一時段有兩門課（例如實驗課分單雙週），需將單週課程放左側、雙週放右側並加上標籤 (`[單周]`, `[雙周]`)。
  - 視覺層級：去除課表格子內的教師名字（移至 Tooltip），上課地點字體適度縮小（11px）並放在底部，課程名稱加粗（13px）。
- **挑戰與修復**：
  - 實作匯出功能時導入了 `html2canvas` 套件，成功讓使用者能一鍵下載 PNG 圖片，並將背景顏色替換為一致的深色主題。
  - 為了讓單雙週並排，曾試圖將表格儲存格 (`td`) 直接設為 `display: flex`，導致了**嚴重的 HTML 表格跑版與高度崩潰**。
  - **最終解決方案**：
    - 將 `display: flex` 改套用於儲存格「內部」動態建立的容器 `coursesContainer`。
    - 若該時段有單雙週重疊，設定 `isSplit = true`，讓內部子區塊擁有 `flex: 1` 與 `min-width: 0` 來均分空間，完美恢復原本的課表結構。
    - 調整排序邏輯，強制將標記為「單」的課程排列在左側。
