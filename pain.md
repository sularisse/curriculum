# 中國醫藥大學 課程排課表生成器 - 實作計劃

## 📌 專案目標
建立一個單一 `.html` 文件，讓使用者透過下拉選單篩選中國醫藥大學的課程資訊，並將查詢結果呈現於視覺化課表中。

---

## 🔍 研究發現

### 學校課程查詢系統分析 (`web1.cmu.edu.tw/courseinfo/`)

經過研究，學校課程系統具有以下特點：

1. **參數編碼機制**：所有下拉選項的 value 都使用 Base64 編碼的加密令牌（如 `synPA3HPXq4=` = 115 學年度）
2. **表單提交方式**：透過 POST 到 `/courseinfo/Home/Courselist`
3. **系所連動**：系所下拉選單透過 AJAX POST 到 `/courseinfo/Home/Get_DeptDropdownList` 動態載入
4. **跨域限制（CORS）**：本地 HTML 無法直接跨域請求學校伺服器，這是主要的技術障礙

## 🏗 實作方案

### 方案：CORS Proxy + 動態查詢

由於跨域限制，HTML 會採用 **CORS Proxy** 方式來間接存取學校 API：

1. 透過公開 CORS proxy（如 `https://corsproxy.io/`）轉發請求到學校伺服器
2. 使用者選擇篩選條件後，系統自動組合參數並發送 POST 請求
3. 解析回傳的 HTML 結果頁面，提取課程表格資料
4. 將課程的星期/節次資訊填入視覺化課表

### 技術細節

1. **篩選器**：使用下拉選單（`<select>`），選項 value 直接映射學校系統的編碼值
2. **系所動態載入**：當學年/學期/學制改變時，透過 proxy 呼叫 `Get_DeptDropdownList` API 取得系所列表
3. **結果解析**：查詢結果返回 HTML 表格，使用 DOMParser 解析並提取課程資訊
4. **課表渲染**：將課程依據星期/節次填入格子，並以不同顏色區分課程
5. **設計風格**：深色模式、毛玻璃效果、現代化 UI
